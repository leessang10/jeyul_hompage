# Home Fullscreen Sections Design

**Date:** 2026-04-08

**Goal**

메인 페이지 첫 번째 히어로 섹션은 유지하고, 그 아래의 홈 섹션들을 전면 제거한 뒤 사용자가 제공한 3장의 레퍼런스 화면을 기준으로 풀스크린 단위 섹션 3개로 재구성한다.

**Context**

- 홈 조립 구조는 [`app/page.tsx`](/Users/isangmu/WebstormProjects/01_leessang10@naver.com/jeyul_hompage/app/page.tsx)에 있다.
- 현재 첫 히어로 아래에는 `HomeProjectStoryRail`, `HomeProcessHighlight`, `HomeSplitEntry`, `HomeTrustStrip`, `HomeFinalCta`가 순차적으로 렌더링된다.
- 사용자는 첫 히어로는 유지하고, 그 아래 섹션은 전부 교체하길 원한다.
- 이미지 자산은 실제 사진 대신 플레이스홀더를 사용하고, 카피 문구는 첨부 레퍼런스를 그대로 재현하길 원한다.

**Approved Direction**

1. 첫 히어로 섹션은 변경하지 않는다.
2. 히어로 아래는 3개의 신규 섹션으로 교체한다.
3. 데스크톱 기준 각 섹션은 화면 전체 높이(`100svh`)를 차지한다.
4. 데스크톱에서는 `scroll-snap`을 적용해 한 섹션씩 읽히는 흐름을 만든다.
5. 모바일에서는 같은 콘텐츠를 유지하되, 잘림을 피하기 위해 높이 제약과 내부 배치를 완화한다.
6. 1번 섹션은 4열 균등 분할 레이아웃으로 구성한다.
7. 2번 섹션은 어두운 배경, 대형 카피, 원형 다이어그램, 우측 흰색 솔루션 원형 블록으로 구성한다.
8. 3번 섹션은 배경 비주얼 위 대형 카피와 CTA, 하단 블루 정보 밴드가 한 화면 안에서 마감되게 구성한다.

**Information Architecture**

- 섹션 1
  - 4개 균등 컬럼
  - 각 컬럼 배경 플레이스홀더
  - 타이틀 `설계`, `건설`, `개발`, `인테리어`
  - 각 항목 설명 카피
- 섹션 2
  - 소형 상단 설명 카피
  - 메인 헤드라인 `개발부터 설계·건설까지 완벽한 일관성을 제공합니다`
  - CTA 버튼 `통합 건축 솔루션 문의하기`
  - 중앙 원형 3개 `설계`, `건설`, `개발`
  - 우측 강조 원형 `통합 건축 솔루션 / 시간 단축, 비용 절감`
- 섹션 3
  - 대형 헤드라인 `단순한 건축이 아닌 자산 가치 상승까지 원하시나요?`
  - 서브카피
  - CTA 버튼 `프로젝트 문의하기`
  - 하단 블루 밴드형 푸터 정보

**Visual System**

- 전체 인상은 레퍼런스처럼 건축 회사 사이트의 묵직하고 정제된 톤으로 맞춘다.
- 이미지가 없는 구간은 그레이, 블루, 차콜 계열의 그라데이션과 패턴을 섞은 플레이스홀더 배경으로 처리한다.
- 타이포그래피는 현재 사이트 시스템 폰트를 유지하되, 자간과 크기를 조정해 레퍼런스의 압축감 있는 헤드라인 인상을 만든다.
- 1번 섹션은 텍스트 가독성을 위해 각 컬럼에 어두운 오버레이를 깐다.
- 2번 섹션은 차콜 배경과 얇은 원형 아웃라인으로 구조감을 만든다.
- 3번 섹션은 상단 비주얼과 하단 브랜드 밴드가 자연스럽게 이어지도록 처리한다.

**Behavior**

- 데스크톱 홈 메인 래퍼에 `scroll-snap-type: y mandatory`를 적용한다.
- 각 신규 섹션에 `scroll-snap-align: start`를 부여한다.
- 원형, 컬럼, CTA는 정적 UI 중심으로 구현한다.
- 별도 애니메이션은 최소화하고, 필요 시 진입 시점의 페이드/트랜지션 정도만 사용한다.

**Data / Content**

- 카피는 레퍼런스에 보이는 문구를 그대로 사용한다.
- 이미지 대신 CSS 기반 플레이스홀더 또는 프로젝트 내 재사용 가능한 장면형 배경을 사용한다.
- 하단 회사 정보는 실제 사이트의 브랜드명과 연락처 체계를 유지하되, 레퍼런스의 배치 방식을 따른다.

**Testing / Verification**

- 현재 프로젝트에는 별도 테스트 러너가 없으므로 UI 교체 작업에 대한 단위 테스트는 추가하지 않는다.
- 정적 검증으로 `npm run lint`와 `npm run build`를 실행한다.
- 구현 후 데스크톱/모바일 레이아웃을 직접 확인해 각 섹션의 화면 점유와 순서를 검토한다.

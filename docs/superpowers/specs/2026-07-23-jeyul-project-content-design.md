# 제율디앤씨 프로젝트 콘텐츠 통합 설계

**작성일:** 2026-07-23

## 1. 목표

`프로젝트별 개요 정리.pptx`, `홈페이지 참고 자료.pptx`, 변환된 WEBP 폴더를 현재 Next.js 사이트에 통합한다.

- WEBP 프로젝트 폴더 40개를 전부 포트폴리오에 등록한다.
- 프로젝트 이미지 228개를 프로젝트별 상세 페이지에 연결한다.
- `홈페이지 사진 정리` 폴더의 이미지 14개는 포트폴리오가 아니라 메인과 사업영역 화면에 사용한다.
- 메인에는 전체 프로젝트 중 대표 프로젝트 8개만 노출한다.
- 기존 임시 그래픽과 임시 프로젝트 데이터를 실제 프로젝트 콘텐츠로 교체한다.
- PPT에 표시된 상담 화면 요구사항을 현재 문의 페이지에 반영한다.

## 2. 확정된 구현 방식

정적 프로젝트 매니페스트와 자동 자산 가져오기 스크립트를 사용한다.

- 프로젝트 메타데이터는 Git으로 관리되는 TypeScript 파일에 명시한다.
- 다운로드 폴더는 로컬 원본 보관소로만 취급한다.
- 배포 결과물이 다운로드 폴더의 절대 경로에 의존하지 않게 한다.
- 가져오기 스크립트가 한글 폴더명과 번호형 파일명을 배포용 영문 경로로 변환한다.
- 원본 WEBP 파일은 수정하거나 덮어쓰지 않는다.
- CMS와 데이터베이스는 추가하지 않는다.
- 런타임 폴더 스캔은 사용하지 않는다.

## 3. 현재 코드 기준

현재 프로젝트는 Next.js 16 App Router, React 19, TypeScript, Tailwind CSS 4를 사용한다.

기존 라우트:

- `/`
- `/portfolio`
- `/portfolio/[slug]`
- `/process`
- `/residential`
- `/commercial`
- `/about`
- `/contact`

`/residential`과 `/commercial`은 `/portfolio`의 필터 상태로 이동한다. 이 구조는 유지한다.

현재 `lib/site-content.ts`에는 회사 소개, 홈 카피, 기존 임시 포트폴리오 데이터가 함께 들어 있다. 프로젝트 데이터는 별도 파일로 분리한다.

## 4. 자산 범위

확인된 WEBP 자산:

- 최상위 폴더: 41개
- 실제 프로젝트 폴더: 40개
- 홈페이지 전용 자산 폴더: 1개
- 전체 WEBP: 242개
- 프로젝트 WEBP: 228개
- 홈페이지 전용 WEBP: 14개
- 전체 용량: 약 203MB

`홈페이지 사진 정리`는 프로젝트로 등록하지 않는다.

홈페이지 전용 자산 구성:

- 메인 페이지 사진 8개
- 설계·건설·개발·인테리어 키워드 이미지 4개
- 추가 배경 이미지 2개

일부 WEBP는 파일 크기가 크므로 가져오기 과정에서 배포용 파생본을 생성한다.

- 카드 썸네일: 최대 너비 960px
- 상세 갤러리: 최대 너비 2000px
- 형식: WEBP 유지
- 원본 비율 유지
- 확대 금지
- EXIF 방향 보정
- 원본 파일 보존

이미지 자체를 사람이 선별하지 않는다. 각 폴더의 자연 정렬 기준 첫 이미지를 카드 대표 이미지로 사용한다. 대표 이미지 변경은 프로젝트 매니페스트의 `coverIndex`로만 제어한다.

## 5. 프로젝트 데이터 모델

프로젝트 데이터는 `lib/projects.ts`에 둔다.

```ts
export type ProjectType = "residential" | "commercial";

export type ProjectCategory =
  | "corporate"
  | "model-house"
  | "residential"
  | "education"
  | "retail";

export type ProjectImage = {
  src: string;
  width: number;
  height: number;
  alt: string;
};

export type Project = {
  slug: string;
  sourceFolder: string;
  title: string;
  client?: string;
  type: ProjectType;
  category: ProjectCategory;
  location?: string;
  year?: string;
  completedAt?: string;
  spaces: string[];
  summary: string;
  coverIndex: number;
  featuredOrder?: number;
  images: ProjectImage[];
};
```

규칙:

- `slug`는 영문 소문자와 하이픈만 사용한다.
- `sourceFolder`는 가져오기 스크립트가 원본 폴더를 찾을 때만 사용한다.
- 날짜가 월까지 확인되면 `completedAt`을 `YYYY-MM`로 기록한다.
- PPT에 없는 값은 추측하지 않고 선택 필드로 비워 둔다.
- PPT 오타는 그대로 노출하지 않는다.
- 회사명 표기는 사이트 브랜드 기준인 `JEYUL D&C`로 통일한다.
- `featuredOrder` 1부터 8까지 지정된 프로젝트만 메인에 노출한다.

## 6. 분류 체계

상위 필터:

- 전체
- 주거
- 기업·시설

세부 분류:

- 기업·업무 공간
- 모델하우스
- 주거
- 교육·보육
- 상업

기존 `/portfolio?type=residential`과 `/portfolio?type=commercial` 링크는 유지한다. 세부 분류는 `category` 쿼리로 추가한다.

예:

- `/portfolio?type=commercial&category=corporate`
- `/portfolio?type=residential&category=model-house`

모델하우스는 주거 성격이므로 상위 타입을 `residential`로 둔다. 기업·업무, 교육·보육, 상업은 `commercial`로 둔다.

## 7. 메인 대표 프로젝트

대표 프로젝트는 날짜 정렬 결과에 맡기지 않고 `featuredOrder`로 고정한다.

초기 대표 8개:

1. TKG 태광 김해공장
2. LGC 용성관 식당
3. LGC 대산사택
4. LX 인터내셔널
5. 범양레우스 청량리 모델하우스
6. LX 홀딩스
7. 어반어스 오피스
8. 더 퍼스트 오션뷰 부산 모델하우스

날짜가 없는 신규 폴더가 더 최근 프로젝트로 확인되더라도 자동으로 메인 구성이 바뀌지 않는다.

## 8. 파일 구조

```text
lib/
  projects.ts
  project-query.ts
scripts/
  import-project-assets.mjs
  project-source-map.mjs
public/
  images/
    home/
    services/
    projects/
      <project-slug>/
        thumb-01.webp
        detail-01.webp
components/
  site/
    project-image.tsx
    project-gallery.tsx
    portfolio-card.tsx
    portfolio-grid.tsx
    portfolio-filter-bar.tsx
    home-project-story-rail.tsx
app/
  portfolio/
    page.tsx
    [slug]/
      page.tsx
```

책임:

- `projects.ts`: 정규화된 프로젝트 메타데이터와 이미지 목록
- `project-query.ts`: 필터링, 정렬, 관련 프로젝트 계산
- `project-source-map.mjs`: 원본 폴더와 배포용 slug의 명시적 매핑
- `import-project-assets.mjs`: 이미지 크기 확인, 파생본 생성, 결과 검증
- `project-image.tsx`: `next/image` 공통 설정과 안전한 대체 표시
- `project-gallery.tsx`: 상세 이미지 갤러리

## 9. 홈페이지

PPT 요구사항을 다음처럼 반영한다.

### 메인 히어로

- `홈페이지 사진 정리/메인 페이지 사진`의 8개 WEBP를 순환 배경으로 사용한다.
- 한 번에 한 장만 보인다.
- 자동 전환과 이전·다음 조작을 지원한다.
- `prefers-reduced-motion`에서는 자동 전환을 중지한다.
- 첫 이미지는 우선 로드하고 나머지는 지연 로드한다.
- 기존 영상 경로는 제거하거나 이미지 슬라이더의 대체 경로로 남긴다.

### 사업영역

4개 키워드를 유지한다.

- 설계
- 건설
- 개발
- 인테리어

각 키워드는 이름이 대응되는 WEBP 한 장을 사용한다. PPT에 지정된 문구를 사이트 톤에 맞춰 줄바꿈만 정리하고 의미는 바꾸지 않는다.

### 추가 배경

키워드 폴더의 `5..webp`, `6..webp`는 PPT에서 지정한 후속 섹션의 배경으로 사용한다.

### 대표 프로젝트

- `featuredOrder`가 있는 8개 프로젝트만 노출한다.
- 카드 클릭 시 실제 상세 페이지로 이동한다.
- 임시 `ProjectPlaceholder`는 실제 이미지가 없는 예외 상황에만 사용한다.

## 10. 포트폴리오 목록

목록 페이지는 전체 40개 프로젝트를 보여준다.

- 기본 정렬: 확인 가능한 완료일 내림차순
- 날짜 없는 프로젝트: 날짜 있는 프로젝트 뒤에 제목순
- 상위 타입 필터와 세부 분류 필터 제공
- 검색 기능은 이번 범위에서 제외
- 필터 상태는 URL 쿼리와 동기화
- 결과 개수 표시
- 잘못된 쿼리는 전체 목록으로 안전하게 복귀
- 카드에는 대표 이미지, 프로젝트명, 분류, 위치, 연도 표시

카드 이미지에는 정확한 프로젝트명이 포함된 대체 텍스트를 사용한다.

## 11. 프로젝트 상세

상세 페이지 구성:

1. 프로젝트명과 분류
2. 대표 이미지
3. 위치, 연도, 클라이언트, 공간 구성
4. 프로젝트 요약
5. 전체 이미지 갤러리
6. 같은 세부 분류의 관련 프로젝트
7. 상담 CTA

정적 생성을 위해 `generateStaticParams`는 프로젝트 40개의 slug를 반환한다.

없는 slug는 `notFound()`로 처리한다.

갤러리는 첫 이미지 우선 로드, 나머지 지연 로드를 사용한다. 초기 구현에서는 모달 라이트박스를 추가하지 않는다.

## 12. 상담 페이지

PPT 요구에 맞춰 데스크톱에서는 2열, 모바일에서는 1열로 구성한다.

왼쪽:

- 상담 접수
- 담당자 검토
- 초기 연락
- 현장 또는 요구사항 확인
- 제안 및 일정 협의

오른쪽 신청서:

- 이름 또는 업체명: 필수
- 연락처: 필수
- 프로젝트 유형: 필수
- 예산 범위: 버튼형 선택
- 평수 범위: 버튼형 선택
- 희망 일정: 선택
- 문의 내용: 선택
- 개인정보처리방침 동의: 필수

예산과 평수는 숫자 입력 필드를 사용하지 않는다.

이번 범위에는 실제 메일 전송, 데이터베이스 저장, CRM 연동을 포함하지 않는다. 폼은 클라이언트 검증과 제출 준비 상태까지만 구현하고, 전송 어댑터를 연결할 자리는 명확히 분리한다.

## 13. 오류 처리

- 원본 폴더 누락: 가져오기 스크립트 실패
- 등록 이미지 0개: 가져오기 스크립트 실패
- 중복 slug: 빌드 전 검증 실패
- 중복 `featuredOrder`: 검증 실패
- 대표 프로젝트가 정확히 8개가 아님: 검증 실패
- 지원하지 않는 파일 확장자: 경고 후 제외
- 손상된 WEBP: 프로젝트명과 파일 경로를 출력하고 실패
- 선택 메타데이터 누락: UI에서 해당 행 숨김
- 이미지 로딩 실패: 비율을 유지한 브랜드 대체 블록 표시
- 잘못된 필터 쿼리: 전체 목록 표시
- 없는 상세 slug: 404
- 필수 상담값 누락: 필드별 오류 문구와 `aria-invalid` 표시

## 14. 성능과 접근성

- 모든 콘텐츠 이미지에 `next/image` 사용
- 카드와 상세 이미지에 정확한 `sizes` 지정
- 메인 첫 배경만 우선 로드
- 나머지 이미지는 지연 로드
- 이미지 비율 공간을 미리 확보해 레이아웃 이동 방지
- 슬라이더 조작 버튼에 접근 가능한 이름 제공
- 키보드로 필터와 슬라이더 조작 가능
- 자동 전환은 포인터 진입, 키보드 포커스, 감소된 모션 설정에서 정지
- 필터 상태와 폼 오류를 색상만으로 전달하지 않음

## 17. 완료 기준

- 40개 프로젝트와 228개 프로젝트 이미지가 실제 포트폴리오에서 접근 가능하다.
- 메인에는 지정된 대표 프로젝트 8개만 보인다.
- 홈페이지 전용 이미지 14개가 히어로, 사업영역, 지정 배경에 연결된다.
- 프로젝트 목록 필터와 40개 상세 라우트가 정상 동작한다.
- 상담 화면이 PPT의 레이아웃과 필수 입력 요구를 반영한다.
- 임시 프로젝트 그래픽은 정상 콘텐츠 경로에서 보이지 않는다.
- 테스트, 린트, 프로덕션 빌드가 통과한다.
